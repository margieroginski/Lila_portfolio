## Design Decisions

### Card Behavior

- **Videos:** clicking a card opens an inline modal that plays the MP4 video  
- **Newsletters:** clicking a card opens the URL in a new tab  
- **LinkedIn Posts:** clicking a card opens the URL in a new tab  
- **Luma Events:** clicking a card opens the URL in a new tab

### Metrics Display

- Show engagement metrics (open rates, reactions, guest counts) on each card  
- Keep them subtle — small text below the description, not the headline

### Card Visuals

- Do not use external placeholder image services  
- For each content category, create a styled background using color blocks, typography, or simple SVG shapes that match the existing portfolio aesthetic:  
  - **Videos:** dark background with a centered play button icon  
  - **Newsletters:** warm off-white background with an envelope or text layout motif  
  - **LinkedIn Posts:** LinkedIn blue (\#0A66C2) background with a subtle post/text motif  
  - **Luma Events:** deep purple or green background with a calendar or event motif

### File Dependencies

- All non-video content lives entirely within `portfolio.html` — no external dependencies  
- Videos are not stored in the repo — they must be hosted externally (Vimeo, YouTube unlisted, etc.) and linked via URL

### Visual Design

- Preserve the existing design, typography, color scheme, and layout from `portfolio.html`  
- Replace only the project content (cards, modals, hero copy, name, stats) with Lila's real work  
- Update the project count in the section header to match the actual number of items
